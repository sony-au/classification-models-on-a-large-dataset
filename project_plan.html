Overview of Problem

This project will focus on the Air Pressure system (APS) of heavy Scania trucks in everyday usage. The APS is utilised in various critical functions, such as braking and gear changes, and its failure renders a truck out of service. The dataset presents a binary classification problem, where the positive class constitutes a specific component failure of the APS and the negative class indicates a failure unrelated to the APS. The prediction modelling objective of this binary classification problem is to minimise false classifications, and therefore the total cost, by correctly identifying and classifying the failure type (APS/non-APS).

Dataset Description

The data has been collected from heavy Scania trucks and sourced from UCI Machine Learning Repository. It consists of 76,000 observations, pre-partitioned into training and test sets of 60,000 and 16,000 examples respectively. There are 171 attributes, the names of which have been anonymised for proprietary reasons. The data types consist of single numerical counters, 7 histogram variables (consisting of bins with different conditions), and one factor attribute for the Class.

The challenges of this dataset are: (1) extreme class imbalance (<2% positive class), (2) large size and the confidential nature of the dataset, (3) the high proportion of missing values (NAs), zeroes and outliers, and (4) the motivation to minimise Total Cost to Scania, by minimising misclassification.

Initial Data Analysis and Data Cleaning

Our exploration has been centred on the training set to avoid information leakage to the test set. The barplot below shows the class imbalance of the training set. The second chart displays the distribution of values of column aa_000, which is endemic of more than 90% of all attributes for this dataset. Here, the majority of values are zeroes, while non-zero values have a high variance. Only 11 attributes have a more linearised distribution like bn_000 and ca_000 below.

The histograms below show the proportion of missing and zero values. Whilst only eight attributes contain more than 60% missing values, 49 columns contain more than 60% zero values. All columns also contain mild (outside 1.5 IQR) and extreme outliers (outside 3 IQR). This is because as stated earlier, more than 90% of attributes are skewed towards zero, with a high variance in non-zero values.

Using a large number of features to train a model can reduce the quality of the model. This is because data points in a large-dimensional space can be very far away from each other, resulting in poor model predictions. Additionally, not all features are truly associated with the response variable and some may be highly correlated with each other. Using these noise or redundant features can lead to overfitting and increase the error on the test data (James, Witten, Hastie, and Tibshirani, 2013).Therefore, the first step in our data cleaning was to remove the NAs and reduce the number of features from 170 to a more reasonable number. First, column cd_000 was removed as it contained only two values (1209600 and NA). Then, 51 columns with more than 60% of missing values were also removed, as the large number of NAs in these columns can introduce bias and lead to deterioration of our models.

We removed 74 features with high Pearson correlations with other features in the dataset (r > 0.7). We then performed forward and backward selection methods (using leaps) to further reduce the number of features to 20 (as shown below, while the optimum number of features is 36, from 20 features onwards, the reduction in Mallows Cp flattened out).

Despite removing the above columns, there are still 39,062 rows remaining with at least one missing value. Three methods were tested to impute these missing values, with examples of scatter plots for az_003 against bk_000 shown below.

As you can see above, Amelia II did not perform well on this dataset as it generated a large number of negative values, likely due to its assumption of multivariate normality (Honaker, King and Blackwell, 2019). The median approach on the other hand, is influenced less by the highly skewed data distribution, but it resulted in many zero values and can cause the uncertainty in the missing value data to be lost (Wishart, 2020). The last approach, MICE, generated more varied values with no negatives because the Classification and Regression Trees (CART) method used in MICE is quite robust against outliers, and can handle both multi-collinearity and skewed distributions (van Buuren, 2018).

Given the high correlation between many pairs of columns in the training dataset, another sensible approach is to use principal component analysis for feature reduction. Since the data is already anonymised, there is no loss of interpretability here as would normally occur with PCA. The scatterplot below shows the two classes plotted with the first two principal components. There is some clear structure to the negative classes, though there is no clear distinction of positive classes. The cumulative variance plot shows over 95% of the variance can be explained using only the first 6 principal components, meaning the number of features can be reduced significantly.

Evaluation Metrics

The correct classification of these failures is paramount from a cost perspective. Scania has provided a cost metric along with the dataset, defined as: Cost = 10FP + 500FN. The reasoning being that a false positive(FP) (false APS fault) requires unnecessary workshop checks keeping trucks off the road, and false negatives(FN) (APS fault not recognized) could cause potential breakdown while in use. Therefore, evaluation metrics such as F1-score and ROC curve showing sensitivity vs specificity will be used to compare and evaluate our models.
